# UPI订阅退款

1. 请求地址:</br>
   https://{domain}/payment/mandate/refund


2. 请求方式:</br>
   POST


3. Content-Type:</br>
   application/json; charset=utf-8


4. 参数说明</br>

|参数名|必选|类型| 说明                                             |
|--|--|--|------------------------------------------------|
|appId|是|string| 接入方应用id                                        |
|subscriptionId|是|string| 创建订阅计划时返回的订阅id                                 |
|orderId|是|string| 需要退款的订单id，在 EXECUTE_MANDATE 回调事件中，回调的orderId的值 |
|signature|是|string| 签名，验签算法请参考```安全说明```文档                         ||


5. 请求参数示例

```json
{
   "appId": "0000000253",
   "subscriptionId": "1990688806181380096",
   "orderId": "1991725759056101376",
   "signature": "ec038a938cc5cd1118690dc56981f579"
}
```


6. 响应数据示例
``` json
{
  "code": 200,
  "message": "success",
  "data": {
    "utr": "532804786712", //该退款交易的utr 
    "orderId": "1991725759056101376", // 退款的订单id
    "refundStatus": 2, // 1退款中 2退款成功 3退款失败
    "merchantOrderId": "1991725758972215296", // 非托管商户发起退款时传的merchantOrderId
    "platMessage": "Your transaction is successful", // 平台退款信息
    "refundId": "1992837637442453504" // 退款id，可用户查看退款状态
  }
```
